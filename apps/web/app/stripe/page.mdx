# Stripe

Stripe API emulation with customers, payment intents, charges, products, prices, and checkout sessions. Includes a hosted checkout page and webhook delivery.

## Customers

- `POST /v1/customers` тАФ create customer
- `GET /v1/customers/:id` тАФ retrieve customer
- `POST /v1/customers/:id` тАФ update customer
- `DELETE /v1/customers/:id` тАФ delete customer
- `GET /v1/customers` тАФ list customers

## Payment Intents

- `POST /v1/payment_intents` тАФ create payment intent
- `GET /v1/payment_intents/:id` тАФ retrieve payment intent
- `POST /v1/payment_intents/:id` тАФ update payment intent
- `POST /v1/payment_intents/:id/confirm` тАФ confirm payment intent
- `POST /v1/payment_intents/:id/cancel` тАФ cancel payment intent
- `GET /v1/payment_intents` тАФ list payment intents

## Charges

- `GET /v1/charges/:id` тАФ retrieve charge
- `GET /v1/charges` тАФ list charges

## Products

- `POST /v1/products` тАФ create product
- `GET /v1/products/:id` тАФ retrieve product
- `GET /v1/products` тАФ list products

## Prices

- `POST /v1/prices` тАФ create price
- `GET /v1/prices/:id` тАФ retrieve price
- `GET /v1/prices` тАФ list prices

## Checkout Sessions

- `POST /v1/checkout/sessions` тАФ create checkout session
- `GET /v1/checkout/sessions/:id` тАФ retrieve session
- `POST /v1/checkout/sessions/:id/expire` тАФ expire session
- `GET /v1/checkout/sessions` тАФ list sessions (filter by `customer`, `status`, `payment_status`)
- `GET /checkout/:id` тАФ hosted checkout page (HTML)
- `POST /checkout/:id/complete` тАФ complete payment flow

## Webhooks

Events are delivered to configured webhook URLs:

- `checkout.session.completed` тАФ when a checkout session is completed
- `checkout.session.expired` тАФ when a checkout session expires
