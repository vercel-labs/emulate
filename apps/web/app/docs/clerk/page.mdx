# Clerk

Clerk authentication and user management emulation with OIDC, users, email addresses, organizations, memberships, invitations, and sessions.

## OAuth / OIDC

- `GET /.well-known/openid-configuration` тАФ OIDC discovery
- `GET /v1/jwks` тАФ JSON Web Key Set (JWKS)
- `GET /oauth/authorize` тАФ authorization endpoint with seeded user picker
- `POST /oauth/token` тАФ authorization code token exchange
- `GET /oauth/userinfo` тАФ user info

## Users

- `GET /v1/users` тАФ list users
- `GET /v1/users/count` тАФ count users
- `POST /v1/users` тАФ create user
- `GET /v1/users/:userId` тАФ get user
- `PATCH /v1/users/:userId` тАФ update user
- `DELETE /v1/users/:userId` тАФ delete user
- `POST /v1/users/:userId/ban` тАФ ban user
- `POST /v1/users/:userId/unban` тАФ unban user
- `POST /v1/users/:userId/lock` тАФ lock user
- `POST /v1/users/:userId/unlock` тАФ unlock user

## Email Addresses

- `POST /v1/email_addresses` тАФ create email address
- `GET /v1/email_addresses/:emailAddressId` тАФ get email address
- `PATCH /v1/email_addresses/:emailAddressId` тАФ update email address
- `DELETE /v1/email_addresses/:emailAddressId` тАФ delete email address

- `PATCH /v1/organizations/:orgId/memberships/:userId` тАФ update membership
