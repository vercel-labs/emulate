# Clerk

Clerk authentication and user management emulation with OIDC, users, email addresses, organizations, memberships, invitations, and sessions.

## OAuth / OIDC

- `GET /.well-known/openid-configuration` ‚Äî OIDC discovery
- `GET /v1/jwks` ‚Äî JSON Web Key Set (JWKS)
- `GET /oauth/authorize` ‚Äî authorization endpoint with seeded user picker
- `POST /oauth/token` ‚Äî authorization code token exchange
- `GET /oauth/userinfo` ‚Äî user info

## Users

- `GET /v1/users` ‚Äî list users
- `GET /v1/users/count` ‚Äî count users
- `POST /v1/users` ‚Äî create user
- `GET /v1/users/:userId` ‚Äî get user
- `PATCH /v1/users/:userId` ‚Äî update user
- `DELETE /v1/users/:userId` ‚Äî delete user
- `POST /v1/users/:userId/ban` ‚Äî ban user
- `POST /v1/users/:userId/unban` ‚Äî unban user
- `POST /v1/users/:userId/lock` ‚Äî lock user
- `POST /v1/users/:userId/unlock` ‚Äî unlock user
