# Microsoft API

The Microsoft emulator provides Microsoft Entra OAuth 2.0 / OpenID Connect plus Microsoft Graph-style Outlook, calendar, and OneDrive surfaces for local development and tests.

## Configuration

```yaml
microsoft:
  users:
    - email: testuser@outlook.com
      name: Test User
  oauth_clients:
    - client_id: emu_microsoft_client_id
      client_secret: emu_microsoft_client_secret
      name: Code App (Microsoft)
      redirect_uris:
        - http://localhost:3000/api/outlook/linking/callback
        - http://localhost:3000/api/outlook/calendar/callback
        - http://localhost:3000/api/outlook/drive/callback
  categories:
    - display_name: Follow Up
      color: preset4
  calendars:
    - id: primary
      name: Calendar
      is_default_calendar: true
```

## Covered Graph Surface

- `/oauth2/v2.0/*` authorization, token, logout, and revoke endpoints
- `/.well-known/openid-configuration` and `/discovery/v2.0/keys`
- `/v1.0/me` and `/v1.0/me/photo/$value`
- `/v1.0/me/messages*` for listing, drafts, replies, forwarding, moving, sending, and attachments
- `/v1.0/me/mailFolders*`, `/v1.0/me/outlook/masterCategories*`, and `/v1.0/me/mailFolders/inbox/messageRules*`
- `/v1.0/subscriptions` and `/$batch`
- `/v1.0/me/calendars` and calendar view endpoints
- `/v1.0/me/drive/*` for folder browsing, simple uploads, downloads, and moves

## Notes

- The message and drive APIs are scoped to the local seeded user and support mutable state during a test run.
- Upload sessions are supported for large Outlook draft attachments.
- The emulator intentionally implements the practical Microsoft Graph subset used by local app flows rather than the entire Graph surface.
